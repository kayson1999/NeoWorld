# 策略生存类游戏 — 策划草案

---

## 一、核心玩法：对话与决策

> 通过对话、用户决策，决定角色性格、行为、命运

### 1.1 被动故事线

- 游戏内预设的故事线，用户去决策决定后续路线
- 用户角色在游戏中作为**决策因子**
- 决策会影响：角色关系、资源走向、阵营声望、生存概率
- 参考：*This War of Mine*（道德困境决策）、*旁观者 Beholder*（监控与举报的灰色抉择）

### 1.2 主动故事线（情感机器人）

- 用户主动输入、模型记忆，决定在游戏世界的命运
- 用户在真实世界的遭遇 → 写入角色
- **记忆系统**：LLM 长期记忆，跨会话追踪用户情感变化，形成角色"命运弧线"
- **情感映射**：现实中的压力/喜悦/孤独 → 游戏中触发对应事件（危机/机遇/社交）

## 二、世界模型

> 定义世界模型（真实世界或者虚拟世界：自己定义）

### 2.1 世界背景 — 多世界可选

参考人类历史，选取若干典型时代背景，玩家进入时选择世界、进入不同的故事：

| 世界 | 背景设定 | 核心矛盾 |
|------|----------|----------|
| **工业围城** | 工业革命时期的工厂城镇 | 老板 vs 工人，工资 / 生存 / 淘汰 |
| **末日方舟** | 灾后废土，资源极度匮乏 | 个体生存 vs 集体利益 |
| **星际殖民** | 外星殖民地，未知环境 | 探索 vs 安全，信任 vs 背叛 |

> 初期可先做 1 个世界打通全流程，后续通过数据配置扩展新世界。

### 2.2 社会结构

- **层级关系**：每个世界有角色层级（如：管理者 → 执行者 → 边缘人）
- **流动规则**：角色可通过决策/资源积累实现阶层升降
- **NPC 生态**：NPC 有自己的生存逻辑，会竞争、合作、背叛

### 2.3 时间与事件

- **回合制 / 日周期**：每"天"为一个决策周期，消耗资源、触发事件
- **阶段演进**：世界有阶段（稳定期 → 动荡期 → 危机期 → 结局），大约 30-50 天一个周期
- **随机事件**：天灾、政策变动、资源发现、NPC 叛变等

## 三、经济与生存模型

### 3.1 资源体系

| 资源类型 | 说明 | 获取方式 |
|----------|------|----------|
| **货币** | 通用交换媒介 | 工作 / 交易 / 任务奖励 |
| **食物** | 生存基础，每日消耗 | 采集 / 购买 / 掠夺 |
| **材料** | 建造/升级所需 | 探索 / 生产 / 交易 |
| **信息** | 隐藏资源，影响决策质量 | 社交 / 探索 / 窃取 |
| **声望** | 社会影响力，解锁高级选项 | 决策积累 / 事件表现 |

### 3.2 经济循环

```
生产（工作/采集） → 资源池 → 消耗（生存开销）
                       ↕
                 交易市场（玩家间/NPC）
                       ↓
               供需动态定价（防通胀/通缩）
```

- 参考：*Frostpunk* 的资源紧缺 + 道德抉择模型
- 关键机制：**资源永远不够用**，迫使玩家做取舍

### 3.3 淘汰与死亡

- 资源归零 → 触发生存危机事件链（乞讨 / 犯罪 / 求助）
- 连续危机未解 → 角色"死亡"（永久或降级重生）
- 多人模式下：被淘汰的玩家可以旁观或以新身份重入

## 四、用户体验与交互

> 增强用户体验、交互：主动、被动 以及 社交

### 4.1 交互方式

| 类型 | 形式 | 说明 |
|------|------|------|
| **被动交互** | 选项决策 | 系统给出 2-4 个选项，玩家选择 |
| **主动交互** | 自由输入 | 玩家用自然语言与 NPC/世界对话，AI 实时响应 |
| **推送交互** | 系统主动触发 | NPC 主动找你、世界事件通知、定时情感问候 |
| **社交交互** | 玩家间互动 | 聊天、交易、结盟、对抗 |

### 4.2 画面与呈现（渐进式）

| 阶段 | 表现形式 |
|------|----------|
| v0.1 | 纯文字 RPG（终端/网页文字界面） |
| v0.5 | 文字 + AI 生成插图（关键场景配图） |
| v1.0 | 2D 像素/卡通场景 + 角色立绘 |
| v2.0 | 可交互 2D 场景地图 + 动画 |

### 4.3 社交系统

- **公共聊天区**：世界广播、区域频道
- **私信与结盟**：玩家间建立信任关系，共享资源/信息
- **竞争排行**：生存天数、财富积累、声望排名
- **协作任务**：需多人合作才能完成的事件

## 五、竞品参考

| 游戏 | 借鉴点 |
|------|--------|
| *This War of Mine* | 战争背景下的道德生存决策、资源极度紧缺的压迫感 |
| *Frostpunk* | 集体生存 vs 个体牺牲、法律/政策系统影响全局 |
| *旁观者 Beholder* | 社会规则下的灰色抉择、监控/举报/包庇的人性博弈 |
| *荒岛的王* | 文字策略生存、选择影响结局 |
| *AI Adventure* | AI 对话驱动叙事、每次游玩独一无二 |
| *AI Dungeon* | 自由输入 + LLM 生成剧情的开放式叙事 |

**NeoWorld 的差异化**：不只是 AI 对话叙事，而是将玩家**真实情感**融入角色命运 + 多人共存的**动态经济生态**。

## 六、难点分析

> 核心难点：**世界模型** 与 **用户体验**

### 1. 世界模型自洽性

- 规则需要逻辑闭环，不能出现"无限刷资源"或"必死局"
- **思路**：用有限状态机 + 数值约束定义规则，反复测试平衡性

### 2. 经济平衡

- 多人环境下容易出现贫富分化、滚雪球效应
- **思路**：动态调节器（资源产出随玩家数/总量自动调节）、阶段性重置机制

### 3. AI 对话质量与一致性

- LLM 容易"出戏"，与世界观矛盾
- **思路**：RAG + 世界观知识库约束生成内容，角色设定作为 System Prompt

### 4. 用户留存

- 文字游戏容易疲劳
- **思路**：推送系统打破被动等待、阶段性剧情高潮、社交驱动、画面渐进升级
